ChartStack by Keen IO

ChartStack is an open-source graphing library and abstraction layer built on top of other graphing libraries. It provides a unified JavaScript API to mask the differences between libraries so you can focus on visualizing your data. Chartstack also extends HTML with new, simple yet powerful charting elements like <piechart> and <barchart>. It allows creating complex charts using static or data-driven data with as minimal coding as possible. Here are some examples:

Chartstack DOM API:

<!-- Chartstack piechart element -->
<piechart title="Top 5 World Populations 2014" width="270px"
height="300px" datasource="piechart.json" library="google"></piechart>
<!-- Google charting library -->
<script src="https://www.google.com/jsapi"></script>
<!-- Chartstack library -->
<script src="chartstack.min.js"></script>

See more examples of the Chartstack DOM API here.

Chartstack JS API:

<!-- Container div -->
<div id="barchart"></div>
<!-- NVD3 charting library -->
<script src="//cdnjs.cloudflare.com/ajax/libs/d3/3.4.3/d3.min.js"></script>
<script src="//cdnjs.cloudflare.com/ajax/libs/nvd3/1.1.14-beta/nv.d3.min.js"></script>
<!-- Chartstack library -->
<script src="js/chartstack.min.js"></script>

<script>

// Chartstack ready method ensures all chart libraries are loaded.
chartstack.ready(function(){
  // Instantiate a new Chart class.
  var linechart = new chartstack.Chart({
    // Target which dom element you want to load your chart in.
    el: document.getElementById('barchart'),
    // Can specify Google Charts or NVD3 charting libraries.
    library: 'nv',
    // Can specify piechart, barchart, or linechart.
    chartType: 'barchart',
    // Can be inline JSON or URL to fetch.
    datasource: 'api/keen/barchart.json',
    // Specify provider if data needs to be transformed.
    provider: 'api.keen.io',
    title: 'User Signups',
    width: '500',
    height: '300'
  });
});

</script>

See more examples of the Chartstack JS API here.

One code: many libraries

Under the hood, ChartStack relies on a charting library but is not fixed to any particular charting library. Google Charts and NVD3 are currently supported with more charting libraries to come.

Includes easy transformation of your data format

We recognize that everybody collects data in different ways and every charting library has it's own arbitrary data format for displaying that data, that's why ChartStack comes with easy methods to normalize incoming data to one unified format. Though there is no real standard for displaying data in charts, once you've setup the normalization plugin for your data source, all charts just work. We've already included a normalization plugin for Keen IO data sources and the community is invited to add theirs in the ChartStack Github repo.

<!-- CSV to JSON transform adapter -->
<script src="fitbit/chartstack.cvs.transform.js"></script>

Change the charting library but not your data

Because Chartstack understands your data format it means switching to a different supported charting library is as easy as a text change without having to change your data to support the other charting api. That way you can focus on building and styling your chart.

Support for advanced users

Chartstack's goal is to be as simple and flexible while not tying you to one Graphing Library. There are cases however, where one graphing library has features that another doesn't. Chartstack keeps the underlying library accessible so that advanced users can still access those features.
